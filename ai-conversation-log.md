### AI（Claude）

`fgets` は「標準入力（キーボード）から1行読み込む」関数です。名前は **f**ile **get** **s**tring（ファイルから文字列を取得）の略。

課題のコードだとこう使ってます:

```c
fgets(next_word, sizeof(next_word), stdin)
```

引数は3つ:

1. `next_word` — 読み込んだ文字を入れる箱（配列）
2. `sizeof(next_word)` — 箱の最大サイズ（これ以上は読まない＝あふれ防止）
3. `stdin` — 読み込み元。`stdin` は「キーボード入力」のこと

## なぜ `\n` が入るのか

あなたが「りんご」と打って **Enter** を押すと、Enter＝改行なので、実際には `りんご` + `改行` がまとめて送られます。`fgets` はその改行もそのまま箱に入れるので、中身は `"りんご\n"` になります。

## なぜ困るのか

しりとりは「前の単語の最後の文字」と「次の単語の最初の文字」を比べます。でも `"りんご\n"` の最後の文字は「ご」じゃなく `\n`（改行）になってしまう。だから比較の前に改行を消しておく、という処理がこれです:

```c
size_t len = strlen(next_word);          // 文字数（正確にはバイト数）を取得
if (len > 0 && next_word[len - 1] == '\n') {  // 末尾が改行なら
    next_word[len - 1] = '\0';           // '\0'（文字列の終わり）に置き換えて消す
}
```

`\0`（ヌル文字）は「ここで文字列おわり」という目印。改行の位置をこれに置き換えると、実質的に改行が消えて `"りんご"` になります。

ちなみに `fgets` を使う理由は、初心者がよく最初に習う `scanf` だとスペースを含む入力や長すぎる入力で事故りやすいからです。`fgets` は「サイズ指定で安全に1行読む」のが強みです。

補足すると、なぜ `printf` で表示に使う「文字列」がこういう仕組みなのか気になったら、そこも説明できます。気軽にどうぞ。

### AI（Claude）

「フラグ変数」は、**状態を覚えておくためのON/OFFスイッチ**のことです。中身は普通の `int` で、`0` を「OFF」、`1` を「ON」として使う、という約束事だけの話。特別な型ではありません。

## なぜ必要か

今の作りは「ゲーム終了」で `break` してループ**そのものを抜けて**しまう。抜けたらもう入力を受け付けないので、終了後にリセットできない。

そこで発想を変えて、**ループは抜けずに回し続ける**。代わりに「今ゲーム中か、終了状態か」をスイッチ（フラグ）で覚えておく。終了しても入力は受け付けて、`reset` が来たらスイッチをONに戻す、という作戦です。

## イメージ

```c
int game_over = 0;   // フラグ。0=プレイ中, 1=終了状態

while (1) {          // ← break しないので、ずっと回る
    // 入力を受け取る（省略）

    // どんな状態でも reset は効くようにする
    if (strcmp(next_word, "reset") == 0) {
        // 履歴を初期化
        history_count = 1;
        strcpy(history[0], "しりとり");
        game_over = 0;              // ← スイッチをプレイ中に戻す
        printf("リセットしました\n\n");
        continue;
    }

    // 終了状態なら、しりとりの処理はスキップ
    if (game_over) {
        printf("ゲームは終了しています。reset で最初から\n\n");
        continue;
    }

    // ここから下は「プレイ中」のときだけ動く処理
    // …しりとり判定…

    // 「ん」で終わったら → break ではなく、フラグを立てる
    if (/* んで終わる */) {
        printf("「ん」で終わり！ゲーム終了\n\n");
        game_over = 1;             // ← break の代わり
        continue;                  // ループには残る
    }
}
```

## ポイントは3つ

`break` を **`game_over = 1;` に置き換える**のが肝。これで「終了したけどループには残る」状態になる。次に、終了状態のときは `if (game_over) { ... continue; }` でしりとり判定を**素通り**させる。最後に、`reset` は状態に関係なく一番先にチェックして、来たら `game_over = 0` に戻す。

こうすると「終了 → reset → また遊べる」が実現します。

補足すると、`while (1)` は「永遠に回れ」なので、本当にプログラムを終わらせる出口（例: `quit` と入力したら `break`、あるいは Ctrl+D の `fgets == NULL`）は別途残しておくと親切です。まずは `game_over` フラグを1個足すところから試してみてください。

## フラグに `int` を使う理由

さっきの `game_over` の話に戻ると、C言語には元々「true / false」を表す専用の型がありません（新しめの書き方では `bool` もありますが、標準では別途 `#include <stdbool.h>` が必要）。そこで昔からの定番として、**整数の `int` を借りて `0`=OFF、`1`=ON として使う**んです。

```c
int game_over = 0;   // 「整数の箱だけど、0か1しか入れない」という運用
```

```c
                next_word[len - 1] = '\0';
                len--;
            }

            if (len == 0) {

            }

            if (strcmp(next_word, "reset") == 0) {

                history_count = 1;
```

## 1. 文字には「番号」がある（コードポイント）

世界中の文字には Unicode で通し番号が振られています。これを**コードポイント**と呼び、`U+3042`（あ）のように書きます。ひらがな・カタカナはちょうど `U+3040`〜`U+30FF` の範囲に固まっています。

## 2. UTF-8 は番号を「バイト」に変換するルール

コンピュータはバイト（0〜255の数）しか扱えないので、コードポイントをバイト列に変換します。その変換方式の一つが **UTF-8**。番号の大きさで使うバイト数が変わります:

- U+0000〜U+007F（英数字）→ **1バイト**
- U+0080〜U+07FF → **2バイト**
- U+0800〜U+FFFF（日本語はここ）→ **3バイト**
- それ以上 → 4バイト

ひらがな・カタカナ（U+3040〜U+30FF）は3バイトの範囲なので、必ず3バイトになります。

## 3. 3バイトのときの「型」が決まっている

3バイトになる文字は、ビットの並びがこう決められています（`x` の部分にコードポイントのビットを詰める）:

```
1110xxxx  10xxxxxx  10xxxxxx
```

先頭バイトは必ず `1110` で始まる、という約束です。ここに実際の番号を当てはめてみます。

## 4. 実際に「あ」(U+3042) で計算

`U+3042` を2進数にすると:

```
0011 0000 0100 0010   (16ビット)
```

これを上の型の `x` に、下位から詰めていきます:

```
1110 xxxx   10 xxxxxx   10 xxxxxx
1110 0011   10 000001   10 000010
= 0xE3      = 0x81       = 0x82
```

先頭バイトが `1110 0011` = **0xE3** になりました。

## なぜ「固定」なのか

先頭バイトの `1110xxxx` の `xxxx`（4ビット）には、コードポイントの**上位4ビット**が入ります。U+3040〜U+30FF はどれも上位が `0011 0000...`（16進で `30xx`）で始まるので、この上位4ビットが**全部同じ `0011`** なんです。

```
1110 + 0011 = 1110 0011 = 0xE3
```

だから、この範囲の文字はどれも先頭バイトが `0xE3` で揃う、というわけです。変わるのは2バイト目・3バイト目だけ（`U+30xx` の下位ビットがそこに入る）。

まとめると、**「先頭バイトが 0xE3 固定」なのは、ひらがな・カタカナのコードポイントが全部 `U+30xx` 台に収まっていて、UTF-8 のビット配置ではその上位部分が先頭バイトに来るから**、です。ちなみに漢字（U+4E00〜など）は上位ビットが違うので、先頭バイトは `0xE4`〜`0xE9` あたりになります。

## やり方：文字を「番号（コードポイント）」に直して比べる

3バイトのUTF-8を元のコードポイント（U+3042 のような番号）に戻す関数を作ります。戻したうえで、カタカナ範囲なら 0x60 を引いてひらがなに寄せる。こうすれば「あ」も「ア」も同じ `0x3042` になります。

```c
// 3バイトのかな文字を番号(コードポイント)に変換。
// カタカナならひらがなに寄せて返す（あ と ア を同じ値にする）
int kana_code(const char *p) {
    unsigned char b0 = (unsigned char)p[0];
    unsigned char b1 = (unsigned char)p[1];
    unsigned char b2 = (unsigned char)p[2];
    // 3バイトから元の番号を組み立てる
    int cp = ((b0 & 0x0F) << 12) | ((b1 & 0x3F) << 6) | (b2 & 0x3F);
    // カタカナ(ァ〜ヶ)なら 0x60 引いてひらがなにする
    if (cp >= 0x30A1 && cp <= 0x30F6) cp -= 0x60;
    return cp;
}
```

```markdown
## 実装した機能の詳細な説明

### 必須機能

**直前の単語の表示**
毎ターン、直前に使われた単語を「前の単語: ◯◯」として表示する。単語は履歴配列 `history` の末尾要素を参照している。
```
